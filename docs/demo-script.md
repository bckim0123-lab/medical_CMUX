# MediSim Orchestrator — 시연 스크립트

> 목표: 8분 안에 "다중 AI 에이전트가 공공의료 데이터로 정책을 자율 발행한다" 를
> 보여주고, 챗봇으로 마무리.

## 한 줄 메시지

> 단일 프롬프트 한 번 → 5개 에이전트가 협업 → B2G 정책 제언서 + 공공데이터 챗봇

## 시연 흐름 (총 ~8분)

### 0. 인트로 (1분)

> "필수 의료 공백은 단순한 의원 부족이 아니라, 영유아 부모가 골든타임 내
> 도달 가능한가의 문제입니다. 우리는 이걸 사람이 데이터를 만지는 대신,
> **5개 AI 에이전트가 공공데이터를 자율 분석**하게 했습니다."
>
> [화면 보여주기 — 분석 시작 누르기 직전]

### 1. 분석 시작 (2분)

좌측 로그 흐름을 짚으며:

> 1. **Data Agent** — Gemini 2.5 Flash 가 직접 `fetch_hira_api(region="서울특별시", specialty="소아청소년과")` 를 결정·호출.
> 2. **Spatial Agent** — Turf.js 로 1,585개 의원의 1.5km 골든타임 버퍼 → union → 서울 영역과 차집합. 취약 면적·자치구 등급 산출.
> 3. **Policy Agent** — 상위 3개 자치구에 신규 센터 / 셔틀 / 원격진료 3종을 시뮬레이션, 9개 대안 생성.
> 4. **Critic Agent** *(NEW)* — 각 정책 안에 대한 리스크·우려 한 문장 추가. "AI가 자기 제안에 반론을 단다."
> 5. **Editor Agent** — 위 모두를 받아 B2G 표준 마크다운 리포트 발행.

`[tool:fetch_hira_api]`, `[tool:calculate_buffer]` 등 색깔 다른 라인 강조:
> "이게 LLM이 결정해서 호출한 도구들입니다. 코드가 아니라 모델이 결정한 인자."

### 2. 결과 시각화 (1분)

지도 줌아웃:
> "자치구별 색상 — 빨강은 영유아 인구 대비 골든타임 외 비율 30% 이상,
> 노랑은 15~30%, 초록은 안전. 상위 3개 자치구 = **은평구·중랑구·도봉구**.
> 외곽 자치구가 일관되게 취약. 의원 분포가 강남·송파·서초에 쏠려있기 때문."

우측 정책 대안 카드 한 번 보여주기.

### 3. What-if 시뮬레이션 (1분) ⭐ 클라이맥스

> "심사위원분께 직접 만져보시라고 하나 선보일게요."

[은평구 신규 센터 카드 클릭] → 지도에 가상 시설 핀 + 새 buffer (초록 hue 추가) → 빨강 영역 줄어듦.

> "신규 센터 1개소 추가만으로 은평구 영유아 커버리지 +41% 회복. 35억 원
> 투자 대비 영향. 셔틀로 가면 7배 ROI지만 효과는 절반."

[다른 카드 클릭으로 비교 — 중랑구 셔틀] → 화면 변화

### 4. Critic 의견 짚기 (1분)

리포트 "비판적 검토" 섹션:
> "Critic Agent 가 단 의견 — '예산 35억 대비 강남·서초 인접 부지 가용 여부
> 검토 필요'. AI 가 자기 안에 의문도 단다. 의사결정자가 통째로 신뢰
> 못 하니까."

### 5. 챗봇 세션 (2분)

좌측 패널 Q&A 탭으로 전환:

> "분석 끝나도 끝이 아닙니다. 지자체 담당자가 추가 질문 가능."

질문 1: **"마포구 vs 강남구 영유아당 의원수 차이?"**
> 챗봇: "강남구 134개 / 영유아 12,098명 = 1개당 90명. 마포구 21개 /
> 8,200명 = 1개당 390명. 강남이 4.3배 밀집. (HIRA + KOSIS 분석 결과)"

질문 2: **"은평구에 신규 센터 vs 셔틀 어느 게 좋아?"**
> 챗봇: "신규 센터 +41% 커버리지, 35억 원 / 셔틀 +23%, 2.5억 원.
> 단가 효율은 셔틀이 약 6배 우위. 단 응급 야간 대응은 센터만 가능.
> (MediSim 분석)"

질문 3 *(거부 시연)*: **"감기약 추천해줘"**
> 챗봇: "공공데이터 기반 답변만 가능합니다. 가능한 질문 예시: 자치구
> 비교 / 취약 지역 / 정책 비교."

### 6. 마무리 (1분)

> "오늘 시연한 건 8분. 실제 보건복지부 정책 결정 회의가 며칠씩 걸리는 일을
> 5개 에이전트가 자동화. PRD에 적은 대로 LangGraph 로 업그레이드하면
> 분기·재시도·human-in-the-loop 까지 확장 가능. **GitHub: bckim0123-lab/medical_CMUX**.
> 끝."

## 시연 후 Q&A 예상

- "왜 LangGraph 안 썼어요?" → "선형 파이프엔 오버킬. 노드 인터페이스가 LangGraph 시그니처와 동형이라 분기·병렬 필요 시 1파일 교체로 이전."
- "Mock 데이터 비율?" → "병원 좌표는 100% 실 HIRA API. 인구는 통계청 시도 단위 합산은 실, 동별 분포는 합성."
- "Gemini 비용?" → "한 분석에 ~10k 토큰 / Flash 모델 / 약 $0.005."
- "확장성?" → "전국 시군구 250개 → 자동 적용 가능. PRD §6 mock-data fallback 으로 API rate limit 우회."
